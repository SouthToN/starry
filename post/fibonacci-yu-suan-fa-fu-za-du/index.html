Fibonacci与算法复杂度

2019-09-09 4 min read

算法复杂度是指算法在编写成可执行程序后，运行时所需要的资源，资源包括时间资源和内存资源。即通常所考虑的时间复杂度和空间复杂度。

对于每一个所设计出的的算法都要从O(N),O(N^2)😮 notation考虑。

code1:

int a =0, b = 0
for (i = 0; i < N; i++ ) {
    a = a + rand()  #  N * 一个操作 = O(N)
}
for (j = 0;i<N/2; j++){
    b = b + rand();  #  N/2 * 一个操作 = 1/2 * O(N) =  O(N)
}

时间复杂度：O(N)
空间复杂度：O(1) //2个单位的内存空间 # constant space complexity 常数级的空间复杂度

code2:

int a  = 0;
for (i = 0; i < N; i++ ) {
    for (j = N; j > i; j-- ) {
        a = a + i + j;
    }
}

操作数
i = 0: j = N ... 1 N
i = 1: j = N ... 2 N-1
i = 2: j = N ... 3 N-2
i = N - 1: j = N N
操作总数 = 1+2+3+...+N = N * (N+1)/2 = N*N/2 +N/2 = 1/2 * O(N^2) + 1/2 * O(N) = O(N^2) + O(N) = O(N^2)
时间复杂度：O(N^2)
空间复杂度：O(1) //3个单位(a,i,j)的内存空间 # constant space complexity 常数级的空间复杂度

code3:

int i,j,k = 0
for (i = N/2; i <= N; i++ ) {
    for (j = 2; j <= N; j = j * 2 ) {
        k = k + N / 2;
    }
}

时间复杂度：O(N*log(N))
空间复杂度：O(1)

code4:

int a = 0, i = N;
while(i>0){
    a += i;  #1个操作
    i /= 2;  #1个操作
}

eg
N = 40; i = 40
操作数
i=20 2
i=10 2
i=5 2
i=2 2
i=1 2
i=0 2
结束
总操作数 = 2 * 6 = 2* log(40)
则
时间复杂度：O(log(N))
空间复杂度：O(1)

定理：如果x的时间复杂度要优于y的市价复杂度，那么假设存在一个足够大的数M，当n>M时，我们可以保证x的实际效率要优于y的实际效率。

code5:

def fib(n):
  if n<3:
    return 1
  return fib(n-2) + fib(n-1)
fib(6)

时间复杂度 ：O(2^N)
对于递归问题求时间复杂度画递归树
递归问题存在大量重复计算，解决方法就是采用动态规划
空间复杂度 ：O(N) （方法栈来计算）

code6:

import numpy as np
def fib(n):
  tmp = np.zeros(n)
  tmp[0] = 1
  tmp[1] = 1
  for i in range(2,n):
    tmp[i] = tmp[i-2] + tmp[i-1]
  return tmp[n-1]

时间复杂度 ：O(N)
空间复杂度 ：O(2^N)

code7:

def fib(n):
  a = 1
  b = 1
  c = 0
  for i in range(2,n):
    c = a + b
    a = b
    b = c
  return c

时间复杂度 ：O(N)
空间复杂度 ：O(1)
